# Big-O Notation

## 부제목
점근적 상한 표기법 — 알고리즘 복잡도의 언어

## 한 줄 정의
Big-O 표기법은 함수의 점근적 상한(asymptotic upper bound)을 나타내며, 입력 크기 `n`이 충분히 클 때 알고리즘의 성장률을 기술하는 수학적 언어다.

## 핵심 개념
**정의:** `f(n) = O(g(n))` ⟺ 양의 상수 `c`와 `n₀`가 존재하여 모든 `n ≥ n₀`에 대해 `f(n) ≤ c · g(n)` 이 성립.

> 상수 계수와 낮은 차수 항은 무시한다. 예: `3n² + 5n + 7 = O(n²)`

## 주요 성장 함수 (빠름 → 느림)
- `O(1)` (상수): 배열 인덱스 접근
- `O(log n)` (로그): 이진 탐색
- `O(n)` (선형): 순차 탐색
- `O(n log n)` (선형로그): Merge Sort, Heap Sort
- `O(n²)` (이차): 삽입 정렬 (최악), 버블 정렬
- `O(2ⁿ)` (지수): Brute-force TSP

## 관련 표기법
- **Ω (오메가)** — 점근적 하한: `f(n) = Ω(g(n))`
- **Θ (세타)** — 정확한 점근적 표기: `O`이자 `Ω`
- **o (소문자 o)** — 엄격한 상한 (등호 미포함)
